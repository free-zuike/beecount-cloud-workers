# 备份重构：从 sync_changes 导出改为 projection 表完整快照

## 问题
当前 `performBackup`（`src/services/backup-executor.ts:200-233`）直接查询 `sync_changes` 表导出原始变更记录，不是完整的账本快照。用户期望和 `/sync/full`（sync.ts:1249-1271）一样的完整快照格式。

## 核心改动

### 文件 1：`src/services/backup-executor.ts`（主要改动）

#### 1a. 替换 `fetchLedgerChanges` → `buildLedgerSnapshot`

当前（line 199-208）：
```typescript
async function fetchLedgerChanges(db, ledgerId) {
  return db.prepare('SELECT ... FROM sync_changes WHERE ledger_id = ?').bind(ledgerId).all();
}
```

改为从 projection 表构建完整快照，复用 `/sync/full` 的查询模式（sync.ts:1249-1271）：
```typescript
async function buildLedgerSnapshot(db, userId, ledgerId) {
  // 1. 查 ledgers 表获取元数据（支持 owner + ledger_members fallback）
  // 2. 并行查 5 个 projection 表 + attachment_files（和 /sync/full 一致）：
  //    - read_tx_projection WHERE ledger_id = ?
  //    - read_account_projection WHERE user_id = ? (user-global)
  //    - read_category_projection WHERE user_id = ? (user-global)
  //    - read_tag_projection WHERE user_id = ? (user-global)
  //    - read_budget_projection WHERE ledger_id = ?
  //    - attachment_files WHERE ledger_id = ? (元数据)
  // 3. 组装为完整快照对象
}
```

返回类型：
```typescript
{
  ledger: { external_id, name, currency, month_start_day },
  snapshot: { items, accounts, categories, tags, budgets },
  attachments: [{ id, sha256, file_name, size_bytes, mime_type, storage_path }]
}
```

#### 1b. 修改 `performBackup` 输出格式

当前 v1.0（raw sync_changes）：
```json
{ "ledger_id": "...", "version": "1.0", "changes": [...] }
```

改为 v2.0（完整快照）：
```json
{
  "backup_time": "...",
  "version": "2.0",
  "ledger": { "external_id": "...", "name": "...", "currency": "CNY", "month_start_day": 1 },
  "snapshot": {
    "items": [...],
    "accounts": [...],
    "categories": [...],
    "tags": [...],
    "budgets": [...]
  },
  "attachments": [{ "id": "...", "sha256": "...", "file_name": "...", "size_bytes": 123, "mime_type": "...", "storage_path": "..." }]
}
```

#### 1c. `performBackup` 签名变更

添加 `userId` 参数（user-global 的 accounts/categories/tags 需要 userId 查询）：
```typescript
export async function performBackup(
  db: D1Database,
  runId: number,
  userId: string,       // 新增
  ledgerId: string,
  remoteConfig: Record<string, string>,
  shouldEncrypt?: boolean,
  r2?: R2Bucket
): Promise<BackupResult>
```

### 文件 2：`src/routes/admin_backup.ts`（调用方更新）

Line 1381：传入 userId
```typescript
// 当前
await performBackup(db, runId, ledgerId || 'global', remoteConfig, shouldEncrypt, c.env.R2);
// 改为
await performBackup(db, runId, userId, ledgerId || 'global', remoteConfig, shouldEncrypt, c.env.R2);
```
userId 从 `c.get('userId')` 获取（已存在于 admin_backup 路由的中间件中）。

### 文件 3：`src/services/backup-scheduler.ts`（调用方更新）

Line 90：传入 userId（从 `schedule.user_id` 获取）
```typescript
// 当前
await performBackup(db, runId, ledger.id, remoteConfig, shouldEncrypt, r2);
// 改为
await performBackup(db, runId, schedule.user_id, ledger.id, remoteConfig, shouldEncrypt, r2);
```

## 不改动的部分
- S3 / R2 / WebDAV / FTP / SFTP 上传逻辑不变
- 加密 / 解密逻辑不变
- restore 端点暂不改（恢复需要新逻辑，后续单独做）
- 前端代码不改
- `calculateNextRun` 不变

4. 检查 snapshot 中包含完整的 accounts/categories/tags/budgets/items
